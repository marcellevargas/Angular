- Para criar um arquivo packaje.json
    rodar o comando npm init para criar
    um arquivo padrão do package.json

- O arquivo tsconfig.json
    é o arquivo de configuração que será levado
    sempre em consideração pelo compilador

- compilerOptions => indica as configurações do 
                       compilador. Indicamos que o 
                       resultado final da compilação 
                       é um código compatível com ES6

- outDir => indica aonde está armazenado o código 
                depois de compilado

- include => indica aonde o compilador deve buscar 
                os arquivos para efetuar o processo

- Para que o compilador execute as funções descritas
  no tsconfig.json adicionamos ao arquivo package.json
  "compile":"tsc" dentro de scripts

- O uso do Node.js é opcional mas é extremamente recomendado

- Podemos passar configurações especiais para o compilador 
  através do arquivo tsconfig.json

- Propriedades das classes precisam ser declaradas no corpo
  da classe (convencionado em cima do constructor)

- Como o meu objetivo é gerar arquivos js a partir dos
  arquivos ts, com a sintaxe correta, no arquivo tsconfig.js
  eu adiciono uma configuração dentro de "compilerOptions"
  chamada "noEmitError":true. Desse modo só serão gerados
  arquivos .js se não houver erros de compilação.
